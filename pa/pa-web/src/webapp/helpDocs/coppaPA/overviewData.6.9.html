Adding On-hold Records
You can add on-hold information for any trial during abstraction.
Before you begin abstracting any trial, be sure you check it out first. Otherwise you will not be able to modify any trial data. For instructions, see Checking In and Checking Out Trials.
How to Add an On-hold Record
1.
Select a trial for abstraction.
2.
On the navigation pane, click On-hold Information.
The On-hold page appears.
3.
Click Add.
The On-hold Information page appears.
4.
In the On-hold Reason Code drop-down list, select the code that indicates the reason that the trial is delayed. The following table lists valid reason codes.
On Hold Code
Description
Submission Incomplete
The following errors are associated with the Submission Incomplete code:
•
Wrong protocol document or IRB approval
•
Outdated IRB approval
•
Missing Informed Consent
•
Missing/incomplete Participating Sites document
Invalid grant
Grant validation failure
The system generates a message to the submitter when this occurs.
Pending disease curation
Disease curation is in process
Pending person curation
Person curation has been requested
Pending organization curation
Organization curation has been requested
Pending intervention curation
Intervention curation has been requested
Other
Any other not listed above reason
5.
Optionally, in the On-hold Description field, type an explanation for putting the trial on hold.
6.
Type the date on which the trial was delayed in the On-hold Date. See Selecting and Entering Dates.
7.
If the trial has continued and is no longer on hold, type the date on which the trial resumed in the Off-hold Date field. See Selecting and Entering Dates.
8.
Click Save.
