Milestone and Processing Status Rules
You can record most milestones multiple times per trial, following the order of the trial submission processing life cycle. However, you can add the following milestones only once per trial submission:
•
Submission Received Date
•
Ready for PDQ Abstraction Date
•
Submission Rejection Date
•
Initial Abstraction Verified Date
•
Initial Submission to http://www.ClinicalTrials.gov
The following rules, among others provided throughout this document apply when recording milestones:
•
If the most current processing status is Submitted, you can only record the Submission Received Date milestone.
•
If the most current processing status is Verification Pending, you can only record the Initial Abstraction Verified Date milestone.
•
If the most current processing status is Abstraction Verified Response or Abstraction Verified No Response, you can record the On-going Abstraction Verified Date milestone.
•
If the most current processing status is Accepted, recording a QC Completed Date milestone sets the trial processing status to Abstracted and Trial Verification Date to the date you record this milestone.
•
If the most current processing status is Verification Pending, recording an Initial Abstraction Verified Date milestone sets the trial processing status to Abstraction Verified Response—if the submitter's feedback on TSR has been received and recorded, or Abstraction Verified No Response—if submitter's feedback has not been received.
•
The Ready for QC Date precedes the QC Start Date.
•
The QC Start Date precedes the QC Completion Date.
•
The TSR Sent Date precedes the Submitter TSR Feedback Date.
•
You cannot record milestones with a future date.
•
Each milestone date must follow the preceding milestone date.
•
Recording the following milestones triggers abstraction validation. You can only record them if the abstraction is verified:
•
Ready for QC Date
•
QC Completed Date
•
Initial Abstraction Verified Date
•
On-going Abstraction Verified Date
•
Trial Summary Report Sent Date
•
If the most current processing status is Verification Pending and the Submitter TSR Feedback Date milestone exists, manually recording the Initial Abstraction Verified Date sets the trial processing status to Abstraction-Verified-Response.
•
If the most current processing status is Verification Pending and the Submitter TSR Feedback Date milestone does not exist, manually recording the Initial Abstraction Verified Date sets the trial processing status to Abstraction-Verified-No Response.
•
If one or more On-hold records exist, if an on-hold state is not terminated, or if the trial is in the Inbox, you cannot record the following milestones:
•
QC Start Date
•
Trial Summary Report Sent Date
•
Submitter Trial Summary Report Feedback Date
•
Initial Abstraction Verified Date
•
Initial Submission to ClinicalTrials.gov
•
On-going Abstraction Verified Date
•
Recording the Trial Summary Report Sent Date milestone triggers sending the XML and TSR in an email message to the submitter.
•
You cannot record the Trial Summary Report Sent Date if the processing status is Submitted, Accepted, or Rejected.
•
Administrative Processing Start Date must precede the Administrative Processing Completed Date
•
Scientific Processing Start Date must precede the Scientific Processing Completed Date
•
You cannot record Scientific Processing Start Date if Administrative Processing Start Date is the most current milestone
•
You cannot record Administrative Processing Start Date if Scientific Processing Start Date is the most current milestone
•
You cannot record Ready for QC Date if both of the following are true:
•
Scientific Processing Completed Date has not been recorded
•
Administrative Processing Completed Date has not been recorded
•
If the most current processing status is Abstracted, recording the Trial Summary Report Sent Date changes the processing status to Verification Pending
•
If the most current processing status is Verification Pending, you can only record the Initial Abstraction Verified Date milestone
When a trial has been amended, the system sends a modified TSR to the submitter that highlights the differences between the original and amended versions.
