Processing Requests for Admin Rights
Each organization registered in the CTRP system can have one primary site administrator (Admin) who is responsible for reviewing and updating user access to trials owned by the organization, and for assigning site administrator status to other users in the organization. In your role as abstractor, you can accept or reject requests for Admin rights from NCI CTRP Registration Site users.
The Inbox lists Admin access requests by default. You can accept or reject these requests on a per-user basis.
How to Grant Admin Rights to NCI CTRP Registration Site Users
1.
On the top section of the navigation pane, click Inbox.
The Pending Admin User page appears. Requests for Site Administration, if any, are listed under Pending Admin User by default.
2.
In the First Name column, click the link for the person to whom you want to accept or reject a request.
The requester’s name and affiliation information is displayed.
3.
Do one of the following:
•
To accept the request, click Accept.
- or-
•
To reject the request, click Reject.
The request is removed from the list of pending Admins, and the user is notified about the acceptance or rejection via email.
