Adding Diseases/Conditions to a Trial
You can link diseases and conditions to a trial by searching for a disease name or condition name from the current list of active records.
Some disease/condition terms become obsolete over time. When you validate the trial, the system displays an error message if the abstraction contains an obsolete term. If this occurs, look up the disease/condition name and replace the obsolete term with the new one.
Before you begin abstracting or validating any trial, be sure you check it out first. Otherwise you will not be able to modify any trial data.
How to Add Diseases/Conditions
1.
On the navigation pane, click Disease/Condition.
The Trial Disease/Condition page appears, with or without any disease or condition records displayed.
2.
Click Add
The Add Disease/Condition page appears.
3.
To search for the name of the disease or condition being studied, click Look Up.
The Search Disease Window appears.
4.
In the Disease Name field, type part or all of the disease/condition being studied.
Type as many letters of the disease/condition name as possible to reduce the number of search results. The system adds wildcards on both sides of the search string (the series of letters you type) for you implicitly. You can type wildcard symbols (*) between characters of the string as necessary.
Examples:
To search for “colon cancer”, type col. The system treats this as if you had searched for all diseases that contain “col” anywhere in the name.
To search for “familial Hodgkin lymphoma”, type fam* lymph. The system treats this as if you searched for all diseases that contain “fam” AND “lymph” in the name.
5.
To include synonyms in your search in addition to preferred names, select the Include Synonym check box.
6.
To limit your search to exact matches only, select the Exact Match Only check box.
7.
Click Search.
The system searches for the disease/condition from both preferred names and synonyms in the database, and returns a list of results.
Note:
The system will not return any results if you have typed too few charac­ters in the Disease Name field or have typed a disease/condition that has numerous sub-concepts.
8.
Scroll through the list (if needed) to locate the disease/condition being studied.
Some records in the results list may not be suitable for selection because they are not high enough in the concept hierarchy.
9.
If the disease/condition being studied appears in the list, click Select, and proceed to Step 10.
10.
If applicable, repeat the steps above to link additional records.
11.
On the Add Disease/Condition page, select the Lead? check box.
You must designate one lead disease/condition in order to be able to complete the abstraction.
12.
If you want to include the disease/condition in the XML document for submission to ClinicalTrials.gov, select the Include in XML? check box.
13.
Click Save.
The record is added to the trial and displayed on the Disease/Condition page.
