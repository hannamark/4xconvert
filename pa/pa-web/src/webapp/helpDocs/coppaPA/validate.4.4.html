Main Steps for Validating Submitted Trials
Before you begin your review and validation of submitted trial data, open or print the protocol, IRB, consent, and/or other trial-related documents so that you can compare trial details in the document(s) to those displayed in Protocol Abstraction. For instructions, see Validating Trial Documents.
To facilitate the comparison, you can view and print the Trial Summary Report (TSR), which contains all data as submitted via the NCI CTRP Registration Site. For instructions, Accessing Trial Summary Reports.
Some proprietary trial submissions may not include a proprietary trial template (protocol document replacement). For such submissions, validate the trial record in ClinicalTrials.gov.
How to Validate Submitted Trials
1.
Select a trial to validate by following the instructions in Selecting Trials that Require Validation.
The trial details are displayed on the Trial Identification page.
2.
On the Trial Identification page, check out the trial. For instructions, see Checking In and Checking Out Trials.
3.
Optionally, in the navigation pane, click View TSR to view and print the Trial Summary Report.
4.
Under the Validation section of the navigation pane, click through each of the Trial Validation menus. Review and compare the trial details in the protocol document or other trial-related documents with those displayed on each page.
5.
Modify the trial details as necessary.
6.
Click Save.
Protocol Abstraction does not automatically save the changes to trial details that you make while validating trials. If you are not able to complete the trial validation process in your current session and want to save the changes you have made so that you can continue with the validation at another time, be sure to save the information before logging out or navigating to another page.
7.
In the navigation pane, click Trial Validation.
8.
Accept or reject the trial by following the instructions in Accepting and Rejecting Trials.
You can also put a trial on hold in case additional information for validation is required. For instructions, see Adding On-hold Records.
