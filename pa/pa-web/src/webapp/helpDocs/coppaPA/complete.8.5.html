Completing and Validating Abstractions
As soon as you click Abstraction Validation on the navigation pane, the Protocol Abstraction system scans the abstracted information for errors and omissions automatically. You must correct all errors before completing the abstraction.
Note:
The instructions below apply to trials with the current trial processing status “Accepted.”
How to Complete and Validate Abstractions
1.
On the navigation pane, click Abstraction Validation.
The Abstraction Validation page appears. If the abstraction contains errors or omissions, they are displayed on this page. Instructions for correcting each error/omission are provided in the Comment column.
2.
If the Abstraction Validation page contains error/omission records, correct them according to the instructions in the Comment column and specific instructions included throughout the online help, and then proceed to the next step.
3.
If the Abstraction Validation page does not contain error/omission records, do one or more of the following to continue:
•
To view the XML version of the trial abstract, click View XML.
- or -
•
To view the Trial Summary Report (TSR), click TSR, and then open the report in a browser (for example, FireFox or Internet Explorer).
The abstraction is verified, and the trial processing status is updated to “Abstraction Verified.” The Abstraction Validation page continues to display options for viewing the XML and TSR reports.
