Accepting and Rejecting Trials
After you establish that all trial information is correct, accept the trial to make it available for future abstraction. If the trial contains incorrect information that you can not modify, or if the trial is a duplicate of another in the CTRP database, reject the trial.
If you determine that a trial is an invalid submission. search for valid submissions through the NCI CTRP website.
Before you accept a trial, validate the following trial characteristics:
•
Distinctiveness. Validate that the trial is not a duplicate of one previously registered and accepted in the CTRP.
•
Appropriateness. For example, the CTRO accepts only interventional trials submitted by a lead organization, and not a participating site in the case of a multi-site trial with an existing coordination center.
•
Completeness. For example, if a given type of document is missing, verify that it is included in another trial document.
•
Accuracy. For example, validate that the principal investigator’s information is correct.
•
Consistency. For example, validate that the Protocol document is consistent with the submitted trial details.
All required fields must contain information in order to be accepted. Additionally, if an XML document is required for submission to ClinicalTrials.gov, ensure that you have provided sponsor, responsible party, and regulatory information.
Before you begin abstracting or validating any trial, be sure you check it out first. Otherwise you will not be able to modify any trial data.
How to Accept or Reject a Trial
1.
In the Action column for the trial you want to validate, click Validate.
You can click either the Action or the NCI Trial Identifier link.
2.
On the Trial Identification page, check out the trial. For instructions, see Checking In and Checking Out Trials.
3.
In the navigation pane, click Trial Validation.
The Trial Validation page displays trial details.
4.
To accept the trial, click Accept.
Although you can accept a trial that requires an XML document for ClinicalTrials.gov without having provided Oversight Authority information, you may receive a warning if you do so.
A message confirms acceptance and the navigation pane displays menus specific to abstracting the trial. The trial processing status changes to “Accepted,” and the system sends a notification of acceptance email message to the trial submitter.
5.
To reject the trial, click Reject.
The Reason For Rejection section appears.
6.
In the Reason For Rejection field, type the reason for rejecting the trial. For example, if you rejected the trial because you found a duplicate record in the system, type Duplicate of record NCI-2008-1234.
7.
Click Done.
A message confirms rejection and the navigation pane displays menus specific to rejecting the trial. The trial processing status changes to “Rejected.”
The system sends a notification of rejection email message to the trial submitter.
