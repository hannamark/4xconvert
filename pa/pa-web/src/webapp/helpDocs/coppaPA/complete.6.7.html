Trial Processing Life Cycle Milestone Events
You can record most milestones multiple times per trial, following the order of the processing life cycle. However, you can add the following milestones only once per trial:
•
Submission Received Date
•
Ready for PDQ Abstraction Date
•
Submission Rejection Date
•
Initial Abstraction Verified Date
•
Initial Submission to CT.GOV Date
The following rules, among others provided throughout this document apply when recording milestones:
•
If the most current processing status is Submitted, you can only record the Submission Received Date milestone.
•
If the most current processing status is Abstracted, you can only record the Initial Abstraction Verified Date milestone.
•
If the most current processing status is Abstraction-Verified, you can record the On-going Abstraction Verified Date' milestone.
•
If the most current processing status is Accepted, recording a QC Completed Date milestone sets the trial processing status to Abstracted.
•
If the most current processing status is Abstracted, recording an Initial Abstraction Verified Date milestone sets the trial processing status to 'Abstraction-Verified'.
•
The Ready for QC Date precedes the 'QC Start Date.
•
The QC Start Date precedes the QC Completion Date.
•
The TSR Sent Date' precedes 'the Submitter TSR Feedback Date'.
•
You cannot record milestones with a future date.
•
Each milestone date must follow the preceding milestone date.
•
Recording the following milestones trigger abstraction validation and can only be recorded if the abstraction is verified:
•
Ready for QC Date'
•
QC Completed Date
•
Initial Abstraction Verified Date
•
On-going Abstraction Verified Date'
